# Этап 1: Подготовка и проектирование

## Задачи этапа

### 1. Выбор предметной области
Необходимо выбрать предметную область, в рамках которой будут осуществляться все дальнейшие работы в течение курса. Это может быть компания, организация, интернет-проект или любая другая сфера деятельности.

**Процесс выбора:**
- Зайдите в таблицу с баллами и оставьте комментарий с вашей темой
- Преподаватель либо занесет тему в таблицу, либо в ответном комментарии объяснит, почему не рекомендует брать данную тему
- Старайтесь не повторяться с темами

### 2. Формулирование функциональных требований
Для выбранной предметной области необходимо сформулировать описание функциональных требований. Они должны полностью описывать поведение вашей системы.

**Рекомендации:**
- После написания функциональных требований получите ревью
- Только после ревью выполняйте оставшиеся пункты (во избежание большого количества переделок в ERD)
- Для получения ревью заполните форму во вкладке "Очередь"

### 3. Проектирование ER-диаграммы
На основе составленных описаний необходимо:
- Выделить сущности, их атрибуты с указанием типов
- Определить связи между сущностями
- Составить ER-диаграмму, которая должна полностью отображать информацию о сущностях

**Требования к описанию атрибутов:**
- Информация о типах данных
- Указание первичных и внешних ключей

**Рекомендация:** Создавать диаграмму в PlantUML.

### 4. Нормализация схемы базы данных
Последним этапом является нормализация схемы базы данных, которая включает:
- Проведение анализа составленной ранее схемы БД для определения функциональных зависимостей
- Рассмотрение каждой таблицы и выявление функциональных зависимостей между ее атрибутами

**Требования:**
- Указать, что все отношения соответствуют 3НФ
- Если при проектировании намеренно использовалась денормализация - это нужно явно указать

## Минимальные требования

- **Количество отношений:** 20+ (принимающий оставляет за собой право добавить функциональные требования для увеличения количества отношений)
- **Нормальная форма:** 3НФ
- **Форма отчетности:** README.md файл, загруженный в свой репозиторий в GitLab
